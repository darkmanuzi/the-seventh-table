THE SEVENTH TABLE — V15.2 WEBHOOK FIX
=====================================

WHY
---
The built-in Netlify form event function was not being triggered after GitHub deploys.
V15.2 uses a normal HTTP POST notification, which Netlify calls after every verified
form submission.

1. DEPLOY THE PATCH
-------------------
Copy this patch into the existing Git project, then commit and push.

2. ADD THE NETLIFY NOTIFICATION
-------------------------------
Netlify project → Project configuration → Notifications → Form submission notifications
→ Add notification → HTTP POST

Event to listen for: New form submission
Form: Any form
URL:
https://the-seventh-table.com/.netlify/functions/form-webhook

Save.

IMPORTANT
---------
Create only ONE HTTP POST notification for Any form. Do not create a separate webhook
for every form, otherwise emails may be sent more than once.

3. ENVIRONMENT VARIABLES
------------------------
Already required:
BREVO_API_KEY
BREVO_LIST_ID

Optional:
BREVO_PARTNERS_LIST_ID=5
BREVO_SENDER_EMAIL=hello@the-seventh-table.com
BREVO_SENDER_NAME=The Seventh Table

4. TEST
-------
Test Guest List and Partners with new email addresses.
Expected:
- Netlify stores the form submission.
- Guest receives the branded welcome email.
- Partner receives the branded acknowledgement.
- Internal partner enquiry arrives at partners@the-seventh-table.com.
- Partner is added to Brevo list #5.

5. LOGS
-------
Netlify → Logs & metrics → Functions → form-webhook
The log now shows the detected form name, fields and Brevo status.
